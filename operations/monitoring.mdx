## Overview

Qdrant provides comprehensive monitoring capabilities through Prometheus-compatible metrics, health check endpoints, and detailed telemetry. This enables you to track performance, detect issues early, and maintain optimal cluster health.

## Metrics Endpoint

Qdrant exposes metrics in Prometheus format at the `/metrics` endpoint.

### Accessing Metrics

```bash
curl http://localhost:6333/metrics
```

Optionally, run metrics on a separate port:

```yaml
# config/config.yaml
service:
  metrics_port: 6334  # Dedicated metrics port
```

## Health Checks

### Liveness Probe

Check if the Qdrant service is running:

```bash
curl http://localhost:6333/
```

**Response:**
```json
{
  "title": "qdrant - vector search engine",
  "version": "1.8.0"
}
```

### Readiness Probe

Check if Qdrant is ready to serve requests:

```bash
curl http://localhost:6333/readyz
```

**States:**
- `200 OK` - Service is ready
- `503 Service Unavailable` - Service is starting or catching up

The readiness probe checks:
1. **Cluster sync** - Node has caught up with cluster consensus
2. **Shard health** - All local shards are healthy and active
3. **Bootstrap** - Node has joined the cluster (if `--bootstrap` was used)

<Note>
The `/readyz` endpoint is particularly useful in Kubernetes for pod readiness probes during rolling updates and cluster formation.
</Note>

### Kubernetes Health Check Example

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: qdrant
spec:
  containers:
  - name: qdrant
    image: qdrant/qdrant:latest
    ports:
    - containerPort: 6333
    livenessProbe:
      httpGet:
        path: /
        port: 6333
      initialDelaySeconds: 10
      periodSeconds: 10
    readinessProbe:
      httpGet:
        path: /readyz
        port: 6333
      initialDelaySeconds: 5
      periodSeconds: 5
      timeoutSeconds: 3
```

## Prometheus Integration

### Prometheus Configuration

```yaml
# prometheus.yml
scrape_configs:
  - job_name: 'qdrant'
    scrape_interval: 15s
    static_configs:
      - targets: ['localhost:6333']
    metrics_path: /metrics
```

### Prometheus Queries

#### Request Rate

```promql
# Total request rate
rate(rest_responses_total[5m])

# Request rate by endpoint
sum(rate(rest_responses_total[5m])) by (endpoint)

# Error rate
sum(rate(rest_responses_total{status!="200"}[5m])) by (endpoint)
```

#### Latency

```promql
# Average latency by endpoint
rate(rest_responses_duration_seconds_sum[5m]) / rate(rest_responses_duration_seconds_count[5m])

# 95th percentile latency
histogram_quantile(0.95, rate(rest_responses_duration_seconds_bucket[5m]))
```

#### Collection Growth

```promql
# Point growth rate
rate(collection_points[5m])

# Vector count by collection
sum(collection_vectors) by (collection)
```

#### Cluster Health

```promql
# Active replicas below minimum
collection_active_replicas_min < 1

# Dead replicas count
collection_dead_replicas > 0

# Cluster working state
cluster_working_state{state="stopped"} == 1
```

#### Resource Usage

```promql
# Memory usage trend
memory_resident_bytes

# Open file descriptors utilization
(process_open_fds / process_max_fds) * 100

# Memory map utilization
(process_open_mmaps / system_max_mmaps) * 100
```

## Telemetry API

For detailed system information, use the telemetry endpoint:

```bash
curl http://localhost:6333/telemetry
```

**Query Parameters:**
- `anonymize` - Anonymize sensitive data (default: false)
- `details_level` - Level of detail (0-2, default: 0)
- `timeout` - Timeout in seconds

Example:
```bash
curl "http://localhost:6333/telemetry?anonymize=true&details_level=1"
```

## Alerting Rules

### Prometheus Alert Examples

```yaml
groups:
  - name: qdrant_alerts
    rules:
      # High error rate
      - alert: HighErrorRate
        expr: |
          sum(rate(rest_responses_total{status!="200"}[5m]))
          /
          sum(rate(rest_responses_total[5m]))
          > 0.05
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High error rate detected"
          description: "Error rate is {{ $value | humanizePercentage }}"

      # Cluster not healthy
      - alert: ClusterUnhealthy
        expr: collection_active_replicas_min < 1
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Cluster has unhealthy shards"
          description: "Minimum active replicas is {{ $value }}"

      # Dead replicas
      - alert: DeadReplicas
        expr: collection_dead_replicas > 0
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "Dead replicas detected"
          description: "{{ $value }} dead replicas found"

      # High memory usage
      - alert: HighMemoryUsage
        expr: memory_resident_bytes > 8e9  # 8GB
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High memory usage"
          description: "Memory usage is {{ $value | humanize }}B"

      # File descriptor exhaustion
      - alert: HighFDUsage
        expr: (process_open_fds / process_max_fds) > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "File descriptor usage high"
          description: "FD usage is {{ $value | humanizePercentage }}"

      # Long-running optimizations
      - alert: LongRunningOptimization
        expr: collection_running_optimizations > 0
        for: 1h
        labels:
          severity: info
        annotations:
          summary: "Long-running optimization detected"
          description: "Optimization running for over 1 hour on {{ $labels.id }}"

      # Consensus stopped
      - alert: ConsensusStoppedWith large update queue
      - alert: LargeUpdateQueue
        expr: collection_update_queue_length > 1000
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "Large update queue"
          description: "{{ $value }} pending operations in {{ $labels.id }}"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Metrics endpoint returns 404">
    Ensure Qdrant is running and accessible. The `/metrics` endpoint is enabled by default.
  </Accordion>

  <Accordion title="High cardinality warnings">
    Qdrant uses whitelisted endpoints to prevent high cardinality. Only key endpoints are tracked in metrics.
  </Accordion>

  <Accordion title="Missing cluster metrics">
    Cluster metrics only appear when cluster mode is enabled (`cluster.enabled: true`).
  </Accordion>

  <Accordion title="Memory metrics show zero">
    Memory metrics use jemalloc stats. Ensure Qdrant is built with jemalloc support.
  </Accordion>
</AccordionGroup>
