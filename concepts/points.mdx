Points are the core data objects in Qdrant. Each point consists of a unique ID, one or more vectors, and an optional payload (metadata).

## Vector Types in Points

### Dense Vectors

```json
{
  "id": 1,
  "vector": [0.875, 0.140625, 0.8976]
}
```

**Source:** `lib/api/src/rest/schema.rs:66-68`

### Sparse Vectors

```json
{
  "id": 2,
  "vector": {
    "indices": [0, 5, 10, 127],
    "values": [0.1, 0.5, 0.8, 0.2]
  }
}
```

Sparse vectors only store non-zero values for efficiency.

### Multi-Dense Vectors

```json
{
  "id": 3,
  "vector": [
    [0.875, 0.140625, 0.1102],
    [0.758, 0.28126, 0.96871],
    [0.621, 0.421878, 0.9375]
  ]
}
```

**Source:** `lib/api/src/rest/schema.rs:70-76`

Used for late interaction models (e.g., ColBERT) where multiple vectors represent one document.

<Tabs>
  <Tab title="Supported Types">

## Point Operations

### Upsert Points

```json
{
  "points": [
    {
      "id": 1,
      "vector": [0.1, 0.2, 0.3],
      "payload": {"city": "Berlin"}
    },
    {
      "id": 2,
      "vector": [0.4, 0.5, 0.6],
      "payload": {"city": "Paris"}
    }
  ]
}
```

Upsert creates new points or updates existing ones.

## Point Retrieval

### Get by ID

```json Request
{
  "ids": [1, 2, 3],
  "with_vector": true,
  "with_payload": true
}
```

```json Response
{
  "result": [
    {
      "id": 1,
      "version": 5,
      "vector": [0.1, 0.2, 0.3],
      "payload": {"city": "Berlin"}
    }
  ]
}
```

### Scroll Points

Iterate through all points:

```json
{
  "limit": 100,
  "with_vector": false,
  "with_payload": {"include": ["city", "price"]}
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Point IDs">
    - Use numeric IDs for sequential data (auto-increment)
    - Use UUIDs for distributed systems
    - Keep IDs immutable after creation
    - Consider UUID v7 for time-ordered IDs
  </Accordion>

  <Accordion title="Payload Design">
    - Index frequently filtered fields
    - Keep payload size reasonable (< 100KB per point)
    - Use nested structures for related data
    - Avoid very deep nesting (> 5 levels)
  </Accordion>

  <Accordion title="Batch Operations">
    - Batch upserts for better performance (100-1000 points)
    - Use batch operations for bulk updates
    - Consider memory limits for large batches
  </Accordion>

  <Accordion title="Vector Management">
    - Ensure vector dimensions match collection config
    - Normalize vectors if using Cosine distance
    - Use named vectors for multi-modal data
    - Validate vector data before upload
  </Accordion>
</AccordionGroup>
