Qdrant supports GPU acceleration for HNSW index construction using Vulkan compute shaders. This can significantly speed up indexing operations for large-scale vector databases.

## Overview

GPU acceleration is implemented in `lib/gpu/` and `lib/segment/src/index/hnsw_index/gpu/` using the Vulkan API for cross-platform GPU compute support.

**Key features:**
- Parallel HNSW graph construction on GPU
- Vulkan-based compute shaders for vector operations
- Automatic CPU fallback when GPU is unavailable
- Support for both NVIDIA and AMD GPUs
- Memory-efficient batched processing

**Performance gains:**
- 2-5x faster index building for large datasets (>100K vectors)
- More consistent indexing times under high load
- Better resource utilization on GPU-equipped servers

## Architecture

### GPU Library Structure

The GPU acceleration layer (`lib/gpu/src/`) provides Vulkan abstractions:

```
lib/gpu/src/
├── lib.rs              # Main GPU interface and error types
├── instance.rs         # Vulkan instance management  
├── device.rs           # GPU device selection and initialization
├── context.rs          # GPU execution context
├── buffer.rs           # GPU memory buffers
├── pipeline.rs         # Compute pipeline management
├── shader.rs           # Shader compilation and loading
├── descriptor_set.rs   # Resource binding
└── allocation_callbacks.rs  # Memory allocation tracking
```

### HNSW GPU Implementation

The HNSW GPU builder (`lib/segment/src/index/hnsw_index/gpu/`) implements parallel graph construction:

```
lib/segment/src/index/hnsw_index/gpu/
├── mod.rs                  # GPU configuration and entry point
├── gpu_graph_builder.rs    # Main GPU graph building logic
├── gpu_level_builder.rs    # Per-level parallel construction
├── gpu_vector_storage/     # GPU-side vector data management
├── gpu_links.rs            # Graph link storage on GPU
├── gpu_visited_flags.rs    # Search state tracking
├── batched_points.rs       # Point batching for GPU processing
└── shader_builder.rs       # Dynamic shader generation
```

## Requirements

### Hardware Requirements

**Minimum:**
- GPU with Vulkan 1.1 support
- 2GB GPU memory (for small datasets)
- PCIe 3.0 x8 connection

**Recommended:**
- Modern GPU (NVIDIA RTX 3000+, AMD RX 6000+)
- 8GB+ GPU memory
- PCIe 4.0 x16 connection

### Software Requirements

**Vulkan Runtime:**
- Vulkan SDK 1.1 or higher
- GPU drivers with Vulkan support

**Linux:**
```bash
# Ubuntu/Debian
sudo apt-get install vulkan-tools libvulkan1

# Verify Vulkan is available
vulkaninfo | grep "deviceName"
```

**Windows:**
- Install GPU drivers from NVIDIA/AMD
- Vulkan runtime is typically included

**Docker:**
```dockerfile
FROM qdrant/qdrant:latest

# Install Vulkan runtime
RUN apt-get update && apt-get install -y \
    vulkan-tools \
    libvulkan1 \
    && rm -rf /var/lib/apt/lists/*

# Enable GPU access (NVIDIA example)
ENV NVIDIA_VISIBLE_DEVICES=all
ENV NVIDIA_DRIVER_CAPABILITIES=compute,utility
```

## Configuration

### Enabling GPU Acceleration

GPU acceleration is controlled at the collection level:

```json
{
  "vectors": {
    "size": 768,
    "distance": "Cosine",
    "hnsw_config": {
      "m": 16,
      "ef_construct": 100,
      "use_gpu": true
    }
  }
}
```

### GPU-Specific Parameters

#### GPU Groups Count

Controls parallel processing capacity (`lib/segment/src/index/hnsw_index/gpu/mod.rs:33`):

```rust
static GPU_GROUPS_COUNT: AtomicUsize = AtomicUsize::new(GPU_GROUPS_COUNT_DEFAULT);
pub const GPU_GROUPS_COUNT_DEFAULT: usize = 512;
```

**Configuration:**
```json
{
  "hnsw_config": {
    "gpu_groups_count": 1024  // Increase for high-end GPUs
  }
}
```

**Tuning guide:**
- Default (512): Good for mid-range GPUs (RTX 3060, RX 6700)
- 1024: High-end GPUs (RTX 3090, A100)
- 2048+: Multi-GPU or very large GPUs (A100 80GB)

#### Half Precision Mode

Use FP16 instead of FP32 for vector operations (`lib/segment/src/index/hnsw_index/gpu/mod.rs:37`):

```rust
static GPU_FORCE_HALF_PRECISION: AtomicBool = AtomicBool::new(false);
```

**Configuration:**
```json
{
  "hnsw_config": {
    "force_half_precision": true  // 2x memory reduction
  }
}
```

**Trade-offs:**
- ✅ 2x GPU memory reduction
- ✅ Faster computation on newer GPUs (RTX 3000+, A100)
- ⚠️ Slightly lower precision (~0.1% recall reduction)

### GPU Timeout

Vulkan operations have a timeout (`lib/segment/src/index/hnsw_index/gpu/mod.rs:29`):

```rust
static GPU_TIMEOUT: std::time::Duration = std::time::Duration::from_secs(60);
```

For large batches, this may need adjustment in source code.

## Performance Tuning

### Batch Size Optimization

GPU processing works in batches (`lib/segment/src/index/hnsw_index/gpu/batched_points.rs`):

```rust
pub struct BatchedPoints {
    // Points grouped by HNSW level for parallel processing
    batches: Vec<Batch>,
}
```

**Optimal batch sizing:**
- Small datasets (less than 10K vectors): GPU may be slower due to overhead
- Medium datasets (10K-100K): 2-3x speedup typical
- Large datasets (more than 100K): 3-5x speedup

### Memory Management

**GPU memory requirements:**
```
GPU Memory = vectors * dimensions * 4 bytes * 3
                       └─ float32    └─ working buffers
```

Example:
- 1M vectors × 768 dimensions × 4 bytes × 3 = ~9.2 GB

**If running out of GPU memory:**
1. Enable half precision:
   ```json
   {"force_half_precision": true}  // Reduces to ~4.6 GB
   ```

2. Reduce batch size (modify source):
   ```rust
   // lib/segment/src/index/hnsw_index/gpu/mod.rs:33
   pub const GPU_GROUPS_COUNT_DEFAULT: usize = 256;  // From 512
   ```

3. Use CPU for initial indexing, GPU for updates

### Monitoring GPU Usage

**NVIDIA GPUs:**
```bash
# Real-time monitoring
watch -n 1 nvidia-smi

# Log GPU usage
nvidia-smi --query-gpu=timestamp,memory.used,utilization.gpu \
           --format=csv --loop=1 > gpu_usage.log
```

**AMD GPUs:**
```bash
# Using radeontop
sudo radeontop
```

## Error Handling

GPU errors (`lib/gpu/src/lib.rs:51`) are handled gracefully:

```rust
pub enum GpuError {
    OutOfMemory,         // GPU/RAM exhausted
    OutOfBounds(String), // Buffer access error
    NotSupported(String),// Feature unavailable
    NotReady,           // Operation pending
    Timeout,            // Operation timeout
    Other(String),      // Other errors
}
```

**Fallback behavior:**
- If GPU initialization fails → CPU indexing used
- If GPU memory exhausted → Error returned, indexing stopped
- If GPU timeout → Operation retried on CPU

Check logs for GPU status:
```bash
# Look for GPU initialization messages
docker logs qdrant 2>&1 | grep -i gpu

# Example output:
# GPU device found: NVIDIA GeForce RTX 3090
# GPU acceleration enabled for HNSW indexing
```

## Troubleshooting

### GPU Not Detected

**Check Vulkan availability:**
```bash
vulkaninfo | head -20
# Should show: Vulkan Instance Version: 1.x.xxx
```

**Install Vulkan SDK:**
```bash
# Ubuntu/Debian  
sudo apt-get install vulkan-tools libvulkan-dev

# Verify
vulkaninfo --summary
```

### Out of Memory Errors

**Error message:**
```
GPU error: OutOfMemory
```

**Solutions:**
1. Reduce GPU groups count:
   ```json
   {"gpu_groups_count": 256}
   ```

2. Enable half precision:
   ```json
   {"force_half_precision": true}
   ```

3. Index in smaller batches (requires code modification)

### Performance Regression

If GPU is slower than CPU:

1. **Check GPU utilization:**
   ```bash
   nvidia-smi dmon  # Should show >80% GPU util
   ```

2. **Verify dataset size:**
   - GPU has overhead for less than 10K vectors
   - Use CPU for small datasets

3. **Check PCIe bandwidth:**
   ```bash
   nvidia-smi topo -m  # Verify PCIe gen and width
   ```

## Best Practices

1. **Use GPU for large-scale indexing:**
   - Enable for collections with >100K vectors
   - Disable for small or frequently updated collections

2. **Monitor GPU memory:**
   - Leave 2-3 GB free for system operations
   - Use half precision for 4K+ dimensional vectors

3. **Optimize batch processing:**
   - Increase `gpu_groups_count` for high-end GPUs
   - Reduce if seeing memory errors

4. **Combine with quantization:**
   ```json
   {
     "hnsw_config": {"use_gpu": true},
     "quantization_config": {
       "scalar": {"type": "int8"}
     }
   }
   ```
   GPU builds index, quantization speeds search.

5. **Use CPU for search:**
   - GPU acceleration doesn't apply to queries yet
   - Optimize search with quantization and SIMD
