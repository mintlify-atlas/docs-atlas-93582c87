# Vector Search Engine for the Next Generation of AI

Qdrant (read: quadrant) is a vector similarity search engine and vector database. It provides a production-ready service with a convenient API to store, search, and manage points—vectors with an additional payload. Qdrant is tailored to extended filtering support, making it useful for all sorts of neural-network or semantic-based matching, faceted search, and other applications.

## Why Qdrant?

**Written in Rust** - Qdrant is built with Rust, making it fast and reliable even under high load. See our [benchmarks](https://qdrant.tech/benchmarks/) for performance comparisons.

**Production-Ready** - With features like write-ahead logging, SIMD hardware acceleration, and async I/O using `io_uring`, Qdrant is designed for demanding production workloads.

**Extended Filtering** - Attach JSON payloads to vectors and filter based on keyword matching, full-text search, numerical ranges, geo-locations, and more. Combine conditions with `should`, `must`, and `must_not` clauses to implement any business logic.

### Python Quick Example

```python
from qdrant_client import QdrantClient

# In-memory instance for testing
qdrant = QdrantClient(":memory:")

# Or persist to disk for prototyping
client = QdrantClient(path="path/to/db")

# Connect to server
client = QdrantClient("http://localhost:6333")
```

## Use Cases

**Semantic Search** - Transform text, images, or other data into vectors and find semantically similar items beyond keyword matching.

**Recommendation Systems** - Build powerful recommendation engines using neural network embeddings with custom filtering logic.

**RAG Applications** - Store and retrieve relevant context for retrieval-augmented generation with LLMs.

**Anomaly Detection** - Identify outliers and unusual patterns in high-dimensional data.

**Image & Video Search** - Enable visual search capabilities by indexing image and video embeddings.

## Architecture Highlights

- **HNSW Indexing** - Hierarchical Navigable Small World graphs for fast approximate nearest neighbor search (m=16, ef_construct=100)
- **Write-Ahead Logging** - Ensures data persistence with 32MB WAL segments
- **Optimizers** - Automatic segment optimization with configurable thresholds (20% deleted threshold, 1000 min vectors)
- **Storage Flexibility** - On-disk payload storage to save RAM while maintaining fast retrieval
- **Performance Tuning** - Configurable search threads, CPU budgets, and concurrent collection loads
