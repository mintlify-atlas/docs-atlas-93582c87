---
title: Create Collection Snapshot
openapi: POST /collections/{collection_name}/snapshots
---

Create a snapshot of a collection. Snapshots capture the complete state of a collection including vectors, payloads, and indexes at a point in time.

## Use Cases

- Create backups before major updates
- Archive collection state for compliance
- Migrate collections between Qdrant instances
- Create test/development copies of production data
- Disaster recovery and point-in-time restoration

## Path Parameters

<ParamField path="collection_name" type="string" required>
  Name of the collection to snapshot.
</ParamField>

## Query Parameters

<ParamField query="wait" type="boolean" default="true">
  If `true`, wait for the snapshot to be created before returning. If `false`, the snapshot is created in the background and the operation returns immediately.
</ParamField>

## Response

<ResponseField name="result" type="object">
  Snapshot description
  
  <ResponseField name="name" type="string">
    Snapshot file name
    
    ```
    my-collection-3766212330831337-2024-07-22-08-31-55.snapshot
    ```
  </ResponseField>
  
  <ResponseField name="creation_time" type="string">
    ISO 8601 timestamp when the snapshot was created
    
    ```
    2024-07-22T08:31:55Z
    ```
  </ResponseField>
  
  <ResponseField name="size" type="integer">
    Snapshot file size in bytes
    
    ```
    1048576
    ```
  </ResponseField>
  
  <ResponseField name="checksum" type="string">
    SHA-256 checksum of the snapshot file for integrity verification
    
    ```
    a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2
    ```
  </ResponseField>
</ResponseField>

<ResponseField name="status" type="string">
  Response status: `ok` or `accepted` (if `wait=false`)
</ResponseField>

<ResponseField name="time" type="number">
  Execution time in seconds
</ResponseField>

## Examples

### Create Snapshot (Synchronous)

```bash
POST /collections/my_collection/snapshots
```

Response:
```json
{
  "result": {
    "name": "my_collection-3766212330831337-2024-07-22-08-31-55.snapshot",
    "creation_time": "2024-07-22T08:31:55Z",
    "size": 1048576,
    "checksum": "a1b2c3d4e5f6..."
  },
  "status": "ok",
  "time": 2.543
}
```

### Create Snapshot (Asynchronous)

```bash
POST /collections/my_collection/snapshots?wait=false
```

Response (returns immediately):
```json
{
  "result": {
    "name": "my_collection-3766212330831337-2024-07-22-08-31-55.snapshot",
    "creation_time": null,
    "size": 0
  },
  "status": "accepted",
  "time": 0.001
}
```

### Using cURL

```bash
curl -X POST \
  'http://localhost:6333/collections/my_collection/snapshots?wait=true'
```

### Using Python Client

```python
from qdrant_client import QdrantClient

client = QdrantClient("http://localhost:6333")

snapshot_info = client.create_snapshot(
    collection_name="my_collection"
)

print(f"Snapshot created: {snapshot_info.name}")
print(f"Size: {snapshot_info.size} bytes")
```

### Using TypeScript Client

```typescript
import { QdrantClient } from "@qdrant/js-client-rest";

const client = new QdrantClient({ url: "http://localhost:6333" });

const snapshot = await client.createSnapshot("my_collection");

console.log(`Snapshot: ${snapshot.name}`);
console.log(`Size: ${snapshot.size} bytes`);
```

## Snapshot Contents

A collection snapshot includes:

- **All vectors** in all configured vector spaces
- **All payload data** for all points
- **All indexes** (payload indexes, vector indexes)
- **Collection configuration** (distance metrics, optimization settings)
- **Shard information** (for distributed collections)

## Snapshot Storage

Snapshots are stored in the configured snapshots directory:

```yaml
# config/production.yaml
storage:
  snapshots_path: /qdrant/snapshots
```

Default paths:
- **Docker**: `/qdrant/snapshots/{collection_name}/`
- **Standalone**: `./storage/snapshots/{collection_name}/`

## Snapshot Naming

Snapshot files follow this naming pattern:
```
{collection_name}-{unique_id}-{timestamp}.snapshot
```

Example:
```
my_collection-3766212330831337-2024-07-22-08-31-55.snapshot
```

## Performance Considerations

- Snapshot creation time depends on collection size
- Large collections may take several minutes
- Use `wait=false` for large collections to avoid timeout
- Snapshots use compression to reduce file size
- Minimal impact on query performance during snapshot creation

### Estimated Times

| Collection Size | Approximate Time |
|----------------|------------------|
| < 100K points | < 1 second |
| 1M points | 5-30 seconds |
| 10M points | 1-5 minutes |
| 100M+ points | 10+ minutes |

## Best Practices

1. **Regular backups**: Schedule periodic snapshots for important collections
2. **Verify checksums**: Always verify snapshot integrity before recovery
3. **Test recovery**: Periodically test snapshot recovery in a staging environment
4. **Monitor disk space**: Snapshots can be large - ensure sufficient storage
5. **Automate cleanup**: Remove old snapshots to manage disk usage
