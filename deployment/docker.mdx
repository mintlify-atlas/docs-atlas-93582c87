## Quick Start

Pull the latest Qdrant image from DockerHub:

```bash
docker pull qdrant/qdrant
```

Run with default configuration:

```bash
docker run -p 6333:6333 qdrant/qdrant
```

<Warning>
This starts an **insecure** deployment without authentication open to all network interfaces. Do not use this configuration in production.
</Warning>

Access the Qdrant dashboard at [http://localhost:6333](http://localhost:6333).

## Production Docker Deployment

For production deployments, use volumes for data persistence and custom configuration:

```bash
docker run -p 6333:6333 -p 6334:6334 \
  -v $(pwd)/qdrant_storage:/qdrant/storage:z \
  -v $(pwd)/qdrant_snapshots:/qdrant/snapshots:z \
  -v $(pwd)/qdrant_config.yaml:/qdrant/config/production.yaml:z \
  --env QDRANT__SERVICE__API_KEY=your-secret-api-key \
  --user=1000:2000 \
  --read-only \
  qdrant/qdrant
```

### Volume Mounts

| Container Path | Purpose | Required |
|----------------|---------|----------|
| `/qdrant/storage` | Vector data and indices | Yes |
| `/qdrant/snapshots` | Backup snapshots | Recommended |
| `/qdrant/config/production.yaml` | Custom configuration | Optional |
| `/qdrant/static` | Web UI static files | Pre-populated |

### Port Mapping

```bash
-p 6333:6333  # HTTP/HTTPS API and Web UI
-p 6334:6334  # gRPC API (optional)
```

### Environment Variables

Override configuration using environment variables with the `QDRANT__` prefix:

```bash
docker run -p 6333:6333 \
  -e QDRANT__SERVICE__API_KEY=your-secret-api-key \
  -e QDRANT__SERVICE__GRPC_PORT=6334 \
  -e QDRANT__SERVICE__ENABLE_TLS=true \
  -e QDRANT__LOG_LEVEL=INFO \
  -v $(pwd)/storage:/qdrant/storage \
  qdrant/qdrant
```

Environment variable format: `QDRANT__<SECTION>__<KEY>=<VALUE>`

Examples:
- `QDRANT__LOG_LEVEL=DEBUG`
- `QDRANT__SERVICE__HTTP_PORT=6333`
- `QDRANT__CLUSTER__ENABLED=true`
- `QDRANT__STORAGE__STORAGE_PATH=/qdrant/storage`

## Docker Compose

### Single Node Setup

Create a `docker-compose.yaml` file:

```yaml
version: '3.7'

services:
  qdrant:
    image: qdrant/qdrant:latest
    container_name: qdrant
    restart: unless-stopped
    ports:
      - "6333:6333"
      - "6334:6334"
    environment:
      - QDRANT__SERVICE__API_KEY=your-secret-api-key
      - QDRANT__SERVICE__GRPC_PORT=6334
      - QDRANT__LOG_LEVEL=INFO
    volumes:
      - ./qdrant_storage:/qdrant/storage:z
      - ./qdrant_snapshots:/qdrant/snapshots:z
    healthcheck:
      test: ["CMD-SHELL", "bash -c ':> /dev/tcp/127.0.0.1/6333' || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3
      start_period: 30s
```

Start the service:

```bash
docker-compose up -d
```

### Multi-Node Cluster

For a 3-node distributed cluster:

```yaml
version: "3.7"

services:
  qdrant_node_1:
    image: qdrant/qdrant:latest
    environment:
      - QDRANT__SERVICE__GRPC_PORT=6334
      - QDRANT__CLUSTER__ENABLED=true
      - QDRANT__CLUSTER__P2P__PORT=6335
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - ./qdrant_storage_1:/qdrant/storage:z
    command: ./qdrant --uri 'http://qdrant_node_1:6335'
    healthcheck:
      test: bash -c ':> /dev/tcp/127.0.0.1/6333' || exit 1
      interval: 5s
      timeout: 5s
      retries: 3

  qdrant_node_2:
    image: qdrant/qdrant:latest
    environment:
      - QDRANT__SERVICE__GRPC_PORT=6334
      - QDRANT__CLUSTER__ENABLED=true
      - QDRANT__CLUSTER__P2P__PORT=6335
    depends_on:
      - qdrant_node_1
    ports:
      - "6433:6333"
      - "6434:6334"
    volumes:
      - ./qdrant_storage_2:/qdrant/storage:z
    command: ./qdrant --bootstrap 'http://qdrant_node_1:6335' --uri 'http://qdrant_node_2:6335'

  qdrant_node_3:
    image: qdrant/qdrant:latest
    environment:
      - QDRANT__SERVICE__GRPC_PORT=6334
      - QDRANT__CLUSTER__ENABLED=true
      - QDRANT__CLUSTER__P2P__PORT=6335
    depends_on:
      - qdrant_node_1
    ports:
      - "6533:6333"
      - "6534:6334"
    volumes:
      - ./qdrant_storage_3:/qdrant/storage:z
    command: ./qdrant --bootstrap 'http://qdrant_node_1:6335' --uri 'http://qdrant_node_3:6335'
```

See [Cluster Setup](/deployment/cluster-setup) for more details.

## GPU Support

Qdrant supports NVIDIA and AMD GPU acceleration for indexing.

### NVIDIA GPU

```bash
docker run --gpus all -p 6333:6333 \
  -e QDRANT__GPU__INDEXING=true \
  -v $(pwd)/storage:/qdrant/storage \
  qdrant/qdrant:latest-gpu-nvidia
```

### AMD GPU

```bash
docker run --device=/dev/kfd --device=/dev/dri -p 6333:6333 \
  -e QDRANT__GPU__INDEXING=true \
  -v $(pwd)/storage:/qdrant/storage \
  qdrant/qdrant:latest-gpu-amd
```

## Building from Source

Build your own Qdrant image:

```bash
git clone https://github.com/qdrant/qdrant.git
cd qdrant
docker build . --tag=qdrant/qdrant:custom
```

### Build Arguments

```bash
docker build \
  --build-arg PROFILE=release \
  --build-arg GPU=nvidia \
  --build-arg FEATURES="web,service" \
  --build-arg TARGET_CPU=native \
  --tag=qdrant/qdrant:custom \
  .
```

Available build arguments:
- `PROFILE`: Cargo profile (`release`, `dev`, `ci`)
- `GPU`: GPU support (`nvidia`, `amd`)
- `FEATURES`: Cargo features to enable
- `TARGET_CPU`: Target CPU optimization
- `LINKER`: Linker to use (`mold`, `lld`)

## Recovery Mode

Qdrant includes automatic recovery mode for OOM situations:

```bash
docker run -p 6333:6333 \
  -e QDRANT_ALLOW_RECOVERY_MODE=true \
  -v $(pwd)/storage:/qdrant/storage \
  qdrant/qdrant
```

If Qdrant is killed during initialization (typically due to OOM), it will restart in recovery mode, allowing you to delete collections to free memory.

## Health Checks

Docker health check example:

```yaml
healthcheck:
  test: ["CMD-SHELL", "bash -c ':> /dev/tcp/127.0.0.1/6333' || exit 1"]
  interval: 10s
  timeout: 5s
  retries: 3
  start_period: 30s
```

Or use the HTTP API:

```yaml
healthcheck:
  test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:6333/"]
  interval: 10s
  timeout: 5s
  retries: 3
```

## Resource Limits

Set memory and CPU limits:

```yaml
services:
  qdrant:
    image: qdrant/qdrant:latest
    deploy:
      resources:
        limits:
          cpus: '4'
          memory: 8G
        reservations:
          cpus: '2'
          memory: 4G
```

## Security Best Practices

1. **Always set an API key** in production:
   ```bash
   -e QDRANT__SERVICE__API_KEY=<strong-secret-key>
   ```

2. **Run as non-root user**:
   ```bash
   --user=1000:2000
   ```

3. **Use read-only filesystem**:
   ```bash
   --read-only
   ```

4. **Enable TLS** for encrypted connections:
   ```bash
   -e QDRANT__SERVICE__ENABLE_TLS=true
   ```

5. **Limit network exposure**:
   ```bash
   -p 127.0.0.1:6333:6333  # Bind to localhost only
   ```

## Troubleshooting

### Container exits immediately

Check logs for errors:
```bash
docker logs qdrant
```

### Permission denied errors

Ensure volume directories have correct permissions:
```bash
mkdir -p qdrant_storage qdrant_snapshots
chown -R 1000:2000 qdrant_storage qdrant_snapshots
```

### Port already in use

Change the host port mapping:
```bash
docker run -p 6444:6333 qdrant/qdrant
```
