# Quick Start

This guide will walk you through creating a collection, inserting vectors with payloads, and performing your first similarity search with filtering. You'll have a working vector search in under 5 minutes.

<Steps>
  <Step title="Start Qdrant with Docker">
    Pull and run the latest Qdrant image from DockerHub:

    ```bash
    docker run -p 6333:6333 -p 6334:6334 qdrant/qdrant
    ```

    <Warning>
      This starts an **insecure deployment** without authentication open to all network interfaces. For production, see [Security Guide](https://qdrant.tech/documentation/guides/security/#secure-your-instance).
    </Warning>

    Qdrant will be accessible at:
    - **REST API**: http://localhost:6333
    - **gRPC API**: http://localhost:6334
    - **Web UI**: http://localhost:6333/dashboard

    <Tip>
      The Web UI provides a built-in interface to explore collections, inspect points, and test queries.
    </Tip>
  </Step>

  <Step title="Create a Collection">
    Create a collection named `test_collection` with 4-dimensional vectors using dot-product distance:

    ```bash
    curl -X PUT 'http://localhost:6333/collections/test_collection' \
      -H 'Content-Type: application/json' \
      --data-raw '{
        "vectors": {
          "size": 4,
          "distance": "Dot"
        }
      }'
    ```

    **Response:**
    ```json
    {
      "result": true,
      "status": "ok",
      "time": 0.031095451
    }
    ```

    **Distance Metrics:**
    - `Dot` - Dot product (higher is more similar)
    - `Cosine` - Cosine similarity
    - `Euclidean` - L2 distance (lower is more similar)
    - `Manhattan` - L1 distance

    <Note>
      Collections are automatically configured with HNSW indexing (m=16, ef_construct=100) and optimization settings (flush_interval_sec=5, deleted_threshold=0.2).
    </Note>
  </Step>

  <Step title="Verify Collection">
    Check that your collection was created successfully:

    ```bash
    curl 'http://localhost:6333/collections/test_collection'
    ```

    You'll see collection details including:
    ```json
    {
      "result": {
        "status": "green",
        "vectors_count": 0,
        "segments_count": 5,
        "config": {
          "params": {
            "vectors": {
              "size": 4,
              "distance": "Dot"
            }
          },
          "hnsw_config": {
            "m": 16,
            "ef_construct": 100,
            "full_scan_threshold": 10000
          },
          "wal_config": {
            "wal_capacity_mb": 32,
            "wal_segments_ahead": 0
          }
        }
      },
      "status": "ok"
    }
    ```
  </Step>

  <Step title="Insert Points with Payload">
    Add vectors with associated payloads. Each point has an ID, vector, and optional JSON payload:

    ```bash
    curl -X PUT 'http://localhost:6333/collections/test_collection/points?wait=true' \
      -H 'Content-Type: application/json' \
      --data-raw '{
        "points": [
          {"id": 1, "vector": [0.05, 0.61, 0.76, 0.74], "payload": {"city": "Berlin"}},
          {"id": 2, "vector": [0.19, 0.81, 0.75, 0.11], "payload": {"city": ["Berlin", "London"]}},
          {"id": 3, "vector": [0.36, 0.55, 0.47, 0.94], "payload": {"city": ["Berlin", "Moscow"]}},
          {"id": 4, "vector": [0.18, 0.01, 0.85, 0.80], "payload": {"city": ["London", "Moscow"]}},
          {"id": 5, "vector": [0.24, 0.18, 0.22, 0.44], "payload": {"count": [0]}},
          {"id": 6, "vector": [0.35, 0.08, 0.11, 0.44]}
        ]
      }'
    ```

    **Response:**
    ```json
    {
      "result": {
        "operation_id": 0,
        "status": "completed"
      },
      "status": "ok",
      "time": 0.000206061
    }
    ```

    <Tip>
      The `wait=true` parameter ensures the operation completes before returning. Points are persisted to the write-ahead log (WAL) immediately.
    </Tip>
  </Step>

  <Step title="Search for Similar Vectors">
    Search for the top 3 most similar vectors:

    ```bash
    curl -X POST 'http://localhost:6333/collections/test_collection/points/search' \
      -H 'Content-Type: application/json' \
      --data-raw '{
        "vector": [0.2, 0.1, 0.9, 0.7],
        "top": 3
      }'
    ```

    **Response:**
    ```json
    {
      "result": [
        { "id": 4, "score": 1.362 },
        { "id": 1, "score": 1.273 },
        { "id": 3, "score": 1.208 }
      ],
      "status": "ok",
      "time": 0.000055785
    }
    ```

    The results are ranked by similarity score (higher is more similar for dot-product).
  </Step>

  <Step title="Search with Filtering">
    Now add a filter to only return points from London:

    ```bash
    curl -X POST 'http://localhost:6333/collections/test_collection/points/search' \
      -H 'Content-Type: application/json' \
      --data-raw '{
        "filter": {
          "should": [
            {
              "key": "city",
              "match": {
                "value": "London"
              }
            }
          ]
        },
        "vector": [0.2, 0.1, 0.9, 0.7],
        "top": 3
      }'
    ```

    **Response:**
    ```json
    {
      "result": [
        { "id": 4, "score": 1.362 },
        { "id": 2, "score": 0.871 }
      ],
      "status": "ok",
      "time": 0.000093972
    }
    ```

    Only points 2 and 4 are returned because they have "London" in their city payload.

    <Note>
      Filters support `should` (OR), `must` (AND), and `must_not` (NOT) clauses. You can also filter by ranges, geo-locations, full-text search, and more.
    </Note>
  </Step>
</Steps>

## Next Steps

    Install the Python client for easier integration:

    ```python
    from qdrant_client import QdrantClient
    from qdrant_client.models import Distance, VectorParams
    
    client = QdrantClient("http://localhost:6333")
    
    client.create_collection(
        collection_name="my_collection",
        vectors_config=VectorParams(size=384, distance=Distance.COSINE),
    )
    ```

    - **Monitoring**: Enable metrics at `/metrics`

## Useful Endpoints

```bash
# List all collections
curl 'http://localhost:6333/collections'

# Get collection info
curl 'http://localhost:6333/collections/test_collection'

# Retrieve a specific point
curl 'http://localhost:6333/collections/test_collection/points/1'

# Delete a collection
curl -X DELETE 'http://localhost:6333/collections/test_collection'

# Health check
curl 'http://localhost:6333/healthz'

# Metrics (Prometheus format)
curl 'http://localhost:6333/metrics'
```

## Common Configuration Options

<CodeGroup>
```yaml config.yaml
service:
  http_port: 6333
  grpc_port: 6334
  
storage:
  storage_path: ./storage
  on_disk_payload: true
  
  hnsw_index:
    m: 16                    # Edges per node (accuracy vs memory)
    ef_construct: 100        # Build-time neighbors
    full_scan_threshold_kb: 10000
    
  performance:
    max_search_threads: 0    # Auto-select based on CPU
    optimizer_cpu_budget: 0  # Auto-select
```

```bash Environment Variables
# Override any config value with environment variables
docker run -p 6333:6333 \
  -e QDRANT__SERVICE__HTTP_PORT=6333 \
  -e QDRANT__SERVICE__GRPC_PORT=6334 \
  -e QDRANT__SERVICE__API_KEY=your_secret_key \
  qdrant/qdrant
```
</CodeGroup>

<Warning>
  **Resource Requirements**: For production workloads, ensure adequate resources:
  - **RAM**: 2GB minimum, 8GB+ recommended for larger datasets
  - **CPU**: 2+ cores recommended
  - **Disk**: SSD recommended for optimal I/O performance (uses `io_uring` on Linux)
</Warning>

## Troubleshooting

**Port already in use:**
```bash
# Map to different ports
docker run -p 6380:6333 -p 6381:6334 qdrant/qdrant
```

**Persist data between restarts:**
```bash
docker run -p 6333:6333 \
  -v $(pwd)/qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

**Enable debug logging:**
```bash
docker run -p 6333:6333 \
  -e QDRANT__LOG_LEVEL=DEBUG \
  qdrant/qdrant
```
